> 数据生成时间：2026-04-19
> 复现种子：`torch.Generator().manual_seed(42)`，用于 `random_split`
> 权重文件：`best_model_ocr.pth`（seed=42 下重训）
> 口径说明：
> - **验证集（val, n=2,549）**：seed=42 从 `dataset/` 划分出的 20%，用于模型选择与调优
> - **独立测试集（held-out, n=1,201）**：`dataset_test/` 经 MD5 内容哈希清洗后得到的真正未见过样本，用于评估泛化能力

### 1.2 独立测试集清洗（`dataset_test/`）

### 1.3 三种口径对比（同一模型、同一权重）

| 口径 | n | Accuracy | 说明 |
|---|---:|---:|---|
| (a) 原始 dataset_test | 1,661 | 0.8001 | 含 MD5 重复与训练污染，**虚高**，仅供参考 |

**(b) 与 (c) 的差 ≈ 4.3 pp**，即 contamination 如不清洗会造成约 4 pp 的虚高。

## 2. 模型与训练配置

## 5. 独立测试集评估（Step 7：`eval_heldout_test.py`）

在 MD5 清洗后真正未见过的 1,201 张 held-out 样本上运行完整 raw 管线（图像→CLIP / 图像→EasyOCR→CLIP 文本 → MLP → argmax）。

| 指标 | 值 |
|---|---:|
| 样本数 | 1,201 |
| **准确率 Accuracy** | **0.7585** |
| OCR 兜底占比 | 86.68% |
| 与验证集差距 | −9.44 pp（0.8529 → 0.7585） |

**解读**：验证集与独立测试集之间存在约 9.4 pp 的差距，主要原因有二：
1. **OCR 兜底率显著升高**（验证错误样本 70.40% → 独立测试集 86.68%），说明 held-out 中文字可检出率更低，跨模态优势被削弱；
2. **领域偏移**：`dataset_test/` 的图像分布与训练集有风格差异（不同来源、不同表情包模板）。

独立测试集准确率 **0.7585** 为论文主要泛化指标；验证集 **0.8529** 为模型选择指标。两者口径不同，不可混用。

## 6. 基线对比（Step 9：`eval_baselines_full.py`）

所有基线在**同一批样本**上评估，保证可比性。

| 基线 | val acc (n=2549) | held-out acc (n=1201) | 与本文差距 (pp, val) |
|---|---:|---:|---:|
| 随机基线（1/5） | 20.00% | 20.00% | −65.29 |
| CLIP Zero-shot（中文 prompt ensemble） | 17.81% | 18.23% | −67.48 |
| CLIP Zero-shot（英文 prompt ensemble） | 56.02% | 55.79% | −29.27 |
| CLIP Linear Probe（冻结图像特征 + LR） | 73.44% | 69.44% | −11.85 |
| **本文方法（CrossModalMLP，图像+OCR 文本）** | **85.29%** | **75.85%** | — |

### 6.1 中文 Zero-shot 低于随机的原因

中文 prompt ensemble 准确率（17.81% / 18.23%）略低于随机基线，原因是 **CLIP 文本编码器对中文语义表示能力弱**，类别嵌入塌缩到「尴尬 / 悲伤」两类：

| 类别 | zh zero-shot (held-out) | en zero-shot (held-out) |
|---|---:|---:|
| 尴尬 | 41.04% | 5.66% |
| 开心 | 3.93% | 78.21% |
| 悲伤 | 40.66% | 50.92% |
| 惊讶 | 0.00% | 55.26% |
| 愤怒 | 4.07% | 79.27% |

中文模板下惊讶类 0 正确、开心 / 愤怒均 <5%，是一种典型的**类别塌缩**现象，说明 CLIP ViT-B/32 在中文情绪词汇上的 prior 严重偏斜。英文 prompt 将准确率拉升到 55~56%，证明 CLIP 多模态能力在英文模板下是可用的，中英差距达 **37.56 pp**。

### 6.2 Linear Probe 与本文方法的差距

CLIP Linear Probe（仅冻结图像特征 + sklearn LogisticRegression）达到 73.44% (val) / 69.44% (held-out)，是当前最强的**单模态图像基线**。本文方法在此之上带来 +11.85 pp (val) / +6.41 pp (held-out) 的提升，**这一提升即跨模态 MLP 引入 OCR 文本模态与非线性组合头的净贡献**。

（held-out 上增益缩小的原因是 OCR 兜底率更高，文本模态信息量减少。）

## 7. 关键论文表述建议（每个实验一段可直接引用的话术）

### 7.1 数据集与划分

> 本文在自建的 5 类中文表情包数据集（尴尬 / 开心 / 悲伤 / 惊讶 / 愤怒，共 12,743 张）上进行实验。为保证结果可复现，使用 `torch.utils.data.random_split` 配合 `torch.Generator().manual_seed(42)` 固定 8:2 划分，得到 10,194 张训练样本与 2,549 张验证样本。另收集 1,661 张外部表情包作为独立测试集 `dataset_test/`，并通过 MD5 文件级哈希清洗剔除 41 张内部重复及 419 张与训练集内容重合的样本，得到 **1,201** 张真正独立 held-out 测试集。

### 7.2 训练

> 图像与 OCR 文本分别由 CLIP ViT-B/32 的图像与文本编码器提取 512 维特征，L2 归一化后送入双分支 CrossModalMLP（每分支 Linear-BN-ReLU-Dropout(0.3) 将 512 维降至 256 维，拼接后经 512→256→128→5 的分类头）。OCR 为空时用占位文本 `"一张表情包"` 兜底。损失为带类别权重的 CrossEntropyLoss（对尴尬类加权 1.5），优化器 Adam (lr=1e-4, wd=1e-4)，调度器 CosineAnnealingLR (T_max=100)，batch size 64，早停 patience 30，于 epoch 121 停止。

### 7.3 验证集性能

> 本文方法在 seed=42 验证集（n=2,549）上取得 **85.29%** 准确率，Macro-F1 **0.8515**，Weighted-F1 **0.8529**。其中尴尬类 F1 最高（0.9009），印证类别加权策略对小样本类的平衡作用；悲伤类 F1 最低（0.8129），与其同其他负性情绪的边界模糊有关（见错误分析）。

### 7.4 错误分析

> 对验证集 375 个错误样本重跑完整原始推理管线并保留图像、OCR 文本、置信度三项证据。分析发现 **70.40%** 的错误样本其 OCR 结果为空，即失败主要集中于"无可辨识文字"的纯表情图像，文本模态缺失直接削弱了跨模态判别能力。此外，混淆对 TOP3 为"悲伤→愤怒"(46)、"愤怒→悲伤"(37)、"开心→尴尬"(30)，其中悲伤 ↔ 愤怒互混合计 83 张（占全部错误的 22.13%），反映出高唤起负性情绪在视觉层面边界模糊。

### 7.5 独立测试集（泛化）

> 为评估模型的真实泛化能力而非对验证集的过拟合，本文在 MD5 哈希清洗后的 1,201 张独立测试集上重跑整条管线，取得 **75.85%** 准确率。对比未清洗的原始 1,661 张（80.01%）与仅内部去重（80.19%），清洗过程剔除了约 **4.3 pp** 的数据污染造成的虚高。val 与 held-out 之间 9.44 pp 的下降主要由 OCR 可检出率下降（70.40% → 86.68% 兜底率）和跨来源视觉分布偏移共同导致。

### 7.6 基线对比

> 本文方法与 4 个基线在同一口径下对比：随机（20.00%）、CLIP Zero-shot 中文 prompt（17.81% / 18.23%）、CLIP Zero-shot 英文 prompt（56.02% / 55.79%）、CLIP Linear Probe（73.44% / 69.44%），本文方法分别在 val / held-out 达到 **85.29% / 75.85%**。
>
> 中文 Zero-shot 低于随机基线的根本原因是 CLIP 文本编码器在中文情绪词汇上 prior 严重塌缩（「惊讶」类 held-out 0.00%），英文 prompt 可将准确率抬升到 55% 以上，说明 CLIP 的跨模态对齐能力在英文语境下可用，但直接使用中文不可行，**跨模态增益必须经过一个针对中文语境学习到的任务头才能释放**——这也是本文引入 OCR 文本→CLIP 文本嵌入→MLP 的动机。相较仅利用图像特征的 Linear Probe，本文方法带来 +11.85 pp (val) / +6.41 pp (held-out) 的净提升，这一差值即跨模态文本融合的纯贡献。
